# 案例資料結構

## 設計原則

- 新欄位只能追加，不刪除未知舊欄位。
- 賽前欄位與賽後欄位分離。
- 每個預測版本有自己的案例 ID、雜湊與報告。
- 舊資料以 `legacy` 標示，不偽裝成 v4 鎖定資料。

## 主要欄位群

| 群組 | 代表欄位 |
|---|---|
| 身分與版本 | 案例ID、資料結構版本、系統版本、研究協議版本、版本序號、取代案例ID |
| 鎖定 | 預測內容雜湊、賽前鎖定時間、鎖定狀態、報告檔案 |
| 比賽 | 比賽、賽事、判斷範圍、體方、用方、賽前偏向 |
| 起卦 | 字數、體卦、用卦、本卦、互卦、動爻、變卦、體用 |
| 足球基線 | 實力、可信度、場地、雙方 λ、三比分、勝平負機率 |
| 卦象修正 | 雙方倍率、上限、修正後 λ、規則狀態快照、診斷 |
| AI | 模型、Prompt、三比分、證據品質、方向信心、控制模式 |
| 賽後 | 實際比分、校準原因、校準摘要、校準狀態、案例品質 |
| 評估 | 各層首選／前三／方向／距離、Brier、LogLoss、是否改善 |

## 雜湊內容

SHA-256 包含：

- 系統與研究協議版本。
- 完整 `MatchInput`。
- 固定 `HexagramResult`。
- 規則預測與完整候選網格。
- 經驗證的 AI 結果（不含供應商原始回應）。
- 使用的相似案例 ID。
- 最終三比分與控制資料。

任何一項改變都會建立新預測版本。

## 遷移

載入舊 CSV 時：

1. 缺少案例 ID 時產生穩定 legacy ID。
2. 缺少校準狀態時依實際比分與理由推定。
3. 舊 `首選／第二／第三比分` 映射到規則與最終欄位。
4. 系統與 schema 標為 `legacy`。
5. 保留所有未知欄位。
