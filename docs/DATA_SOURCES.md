# 賽前資料來源與輸入規範

## 系統目前的資料模式

本專案不會自動抓取比分、新聞、排名或陣容。使用者先在外部查證，再把賽前資訊整理成五個輸入區。這個設計刻意讓資料責任可見，避免搜尋結果中的賽後比分倒灌。

## 可使用的賽前資料

- 官方賽程、比賽時間、場地與賽制。
- 開賽前已公布的名單、停賽、傷停與健康狀態。
- 開賽前世界排名、Elo、近期比賽與對手強度。
- 預計陣型、戰術、定位球、逼搶、反擊與控球特徵。
- 休息天數、旅程、天氣與草皮。
- 小組積分、晉級條件與淘汰賽壓力，但只能採當時已知狀態。

## 禁止輸入賽前流程的資料

- 本場最終或即時比分。
- 本場進球時間、射門、xG、紅黃牌與賽後統計。
- 賽後傷停、晉級結果、下一輪對手。
- 已知淘汰或冠軍結果。
- 為配合已知賽果而修改的文字、實力分或可信度。

## 五個輸入區

| 輸入 | 參與起卦字數 | 用途 |
|---|---:|---|
| 體方段落 | 是 | 體卦 |
| 用方段落 | 是 | 用卦 |
| 完整賽前中性段落 | 是 | 動爻 |
| 賽前補充資料 | 否 | 解卦與 AI 證據 |
| 足球先驗 | 否 | 純足球 λ |

主要三段不應放來源名稱、網址或額外註解，以免無意改變字數。來源與查證摘要應放在補充資料或外部研究紀錄。

## 實力分與可信度

- 實力分 0–100 是中立場基礎實力，不重複加入主場優勢。
- 支持體方不構成加分理由。
- 可信度代表資料新鮮度、完整度與來源一致性，不是勝率或投注信心。
- 資料不足時應降低可信度，而不是用主觀確信補足。
