# 校準規則治理

## 為什麼需要治理

如果每看到一個賽果就建立「某卦＝某比分」，模型一定會在歷史資料看似準確、在新賽事失效。v4 把案例解讀與可執行規則分開。

## 狀態

### `hypothesis`

- 由一場或少量案例提出。
- 可以顯示、討論與回測。
- 預測權重固定為 0。

### `general`

- 來自多案例或預先定義的一般弱先驗。
- 權重 20%。
- 仍受 λ 的 ±25% 總上限限制。

### `reviewed`

- 至少有 3 個互不重複的開發案例支持。
- 條件與效果在留出結果揭曉前已凍結。
- 權重 35%。

### `verified`

- 至少有 10 個未參與規則制定的留出案例。
- 預先指定的主指標改善，且沒有重大校準惡化。
- 審查紀錄包含規則版本、樣本 ID、結果與批准人。
- 權重 100%，但仍受全局上限限制。

## 禁止做法

- 因已知比分調整條件、數字公式或效果大小。
- 同時嘗試大量相加、相減、折半、倍增後挑中答案。
- 把同一場的多段描述當成多個獨立證據。
- 只保存命中規則、不保存失敗案例。
- 在留出樣本結果揭曉後更改該批次的規則版本。
