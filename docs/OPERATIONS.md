# 部署與操作

## 部署前

1. 下載 `data/meihua_cases.csv` 備份。
2. 確認 `.streamlit/secrets.toml` 未被追蹤。
3. 執行編譯、靜態檢查與測試。
4. 在非正式案例上做一次 Streamlit 冒煙測試。

## 部署後檢查

- 頁面顯示 v4.2.0。
- 側邊欄能讀取 GitHub 案例庫。
- 固定起卦後同時顯示足球基線與卦象倍率。
- 倍率均在 0.75–1.25。
- 「卦線劇本」可顯示三時段、五維分數、閘門、數路與候選理由。
- 劇本混合權重介於 0.18–0.46，比分網格機率合計為 1。
- 頁面先顯示本地語義卦線，數值位於「量化審計」。
- AI第一階段顯示盲解主線、替代劇本與反解；第二階段才顯示實力與比分。
- 儲存後 CSV 出現 v4.2 語義、AI盲解與採用劇本欄位。
- 重複儲存相同內容不會新增列。
- 修改實力分後儲存會新增版本並連結前一案例 ID。
- 賽後回填後再次儲存，不會清空實際比分。
- 報告檔名包含內容雜湊，不覆蓋舊報告。

## 備份

- 每次重大版本升級前下載 CSV。
- 定期保留 GitHub repository tag 或 release。
- CSV 與報告都在 Git 歷史中，但 Token 不在 repository。

## 常見故障

### GitHub 讀寫失敗

- 核對 `GITHUB_REPO`、分支與路徑。
- 核對 Token 的 Contents 讀寫權限。
- 檢查 Token 是否過期。
- GitHub 不可用時，App 仍可使用本地暫存，但不要把暫存誤認為已完成遠端鎖定。

### GitHub Models 失敗

- 用側邊欄測試 catalog。
- 核對模型 ID 與 Models 唯讀權限。
- 429 代表額度或速率限制；固定引擎仍可使用。
- 一次完整推理包含兩個模型請求；第一階段成功、第二階段失敗時仍應保留盲解內容。
- 若AI未啟用，頁面應明示目前只有本地語義模板。

### 舊 CSV 顯示異常

- 不要手動刪欄位。
- 從備份恢復後重新啟動 App。
- 用測試複本驗證遷移，再處理正式資料。

## 回滾

程式回滾與案例資料回滾應分開。回滾程式碼時，保留新增的 CSV 欄位；舊程式通常會忽略未知欄位。不要用空白 CSV 覆蓋正式案例庫。
