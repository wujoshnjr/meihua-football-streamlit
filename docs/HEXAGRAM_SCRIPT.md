# 語義卦線與量化劇本 v2

## 目的

舊式比分映射容易把某個卦名或卦數直接套成常見比分。v4.2 先輸出完整語義卦線、主解、反解與失效條件，再把量化層降為審計工具。GitHub Models 的第一階段盲解另有嚴格隔離，見 [BLIND_DELIBERATION.md](BLIND_DELIBERATION.md)。這一層是明示、可測試的研究啟發式，不宣稱是傳統梅花易數中的固定足球規則。

## 時間線

| 階段 | 資訊 | 權重／時窗 |
|---|---|---|
| 開局與主局 | 體、用、本卦、初始五行生剋 | 本卦象意 50% |
| 中段 | 互卦 | 互卦象意 25% |
| 轉折與終局 | 動爻、體用轉象、變卦、終局生剋 | 變卦象意 25% |

動爻剩餘時間權重依初爻到上爻為 `1.00、0.88、0.75、0.62、0.48、0.35`。初爻轉變早、影響時間長；上爻轉變晚，必須保留「追擊形成但未完成」的分支。

## 五個獨立維度

1. `dynamics_score`：節奏、衝擊與事件密度，不直接等同進球。
2. `scoring_channel_score`：禁區缺口、完成手段與防線是否真的被打開。
3. `closure_score`：封鎖、停止、節制與終局收束，是權重而非絕對否決。
4. `volatility_score`：反覆、轉折、失序與生剋方向反轉。
5. `body/use ownership` 與 `body/use finishing`：把能量屬於誰和能否完成破門分開。

本卦、互卦、變卦的標籤只透過預先列出的開放、收束與波動詞彙表計分。五行生剋同時比較初始與終局；方向相反時提高波動並保留後段反轉，不把單一「體剋用」直接判成體勝。

## 劇本閘門

| 閘門 | 主要條件 | 作用 |
|---|---|---|
| 鏡像對消 | 體用同卦，但動能或破門通道不足 | 提高收束，保留 `0-0`／`1-1` |
| 同數共振 | 體用同卦，動能與通道同時達標 | 打開高比分尾部，才允許同數疊加數路 |
| 零球 | 高收束、低通道、低波動，或鏡像對消 | 總球錨點優先 `0、1、2` |
| 高比分 | 綜合開放度達標、大勝成立、強反轉成立或同數共振 | 保留 3+ 與 5+ 分支 |
| 爆發 | 極高開放度，或動能／通道高且收束低，或大勝與開放同向 | 保留 4–7 球分支 |
| 大勝 | 可信足球強弱差、能量集中、通道與強方完成能力同向 | 保留零封大勝與帶一次失球的尾部 |
| 單向破門 | BTTS 不成立，且一方能量與完成能力明顯集中 | 產生 `2-0`、`3-0` 等，而非強迫 `2-1` |

強弱差足夠明確時，收束不能一票否決所有進球；它仍會降低開放度，但零球閘門會被可信的單向優勢或大勝訊號解除。反之，只有動能高而沒有破門通道時，仍可判為高壓但 `0-0`。

## 卦數使用限制

卦數不是進球數，只允許四種次級路徑：

1. 原始體用卦數差在 2–6。
2. 變卦體用卦數差在 2–6。
3. 同數共振時，動方原卦數加變卦數，且轉入震、巽、乾、離等高動能卦。
4. 至少兩條獨立路徑收斂到同一總球數。

數字訊號必須在零球／高比分等環境判斷之後使用。例如雙兌、體方兌轉震且通道已開時，`2+4=6` 可成為六球錨點；若同卦結構缺少破口，則先按鏡像對消處理。

## 從總球到比分

1. 足球 λ 與綜合開放度建立預期總球中心。
2. 零球、高比分、爆發與大勝閘門決定允許的總球錨點。
3. 足球先驗的進球占比與體用能量歸屬混合成分配占比。
4. 兩方完成通道決定 BTTS、單向破門、零封或對攻分支。
5. 關係反轉同時保留「後段翻盤」與「追近未竟」。

固定比分分布為：

$$P_{final}(s)=(1-w)P_{Poisson+pattern}(s)+wQ_{script}(s)$$

其中劇本權重 $w$ 限制在 18–46%；$Q_{script}$ 只在已列出的比分原型上有質量。系統保存每格的基礎機率、劇本機率、最終機率與觸發理由，因此可在賽後分層檢查劇本究竟改善或惡化了足球基線。

## 防止事後擬合

- 本場實際比分不進入起卦、足球先驗、劇本或 AI 賽前 Prompt。
- 單一賽事只能提出 `hypothesis`，預測權重為 0。
- 閘門與門檻按版本凍結；修改後必須建立新版本並重新做留出測試。
- 代表案例測試只驗證泛化結構，例如「同數可對消也可共振」、「強反轉保留雙向一球差」，不以隊名或已知比分硬編碼規則。
- 384 種體卦×用卦×動爻回歸檢查機率守恆、輸出範圍與比分多樣性，但不把結構測試冒充預測準確率。
