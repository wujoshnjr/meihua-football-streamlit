# 評估方法

## 研究問題

1. 卦象有界 λ 修正是否優於純足球先驗？
2. 連續卦象劇本情境是否優於只有 λ 修正的比分網格？
3. GitHub Models 受控審查是否再優於固定劇本？
4. 改善是否能在未參與規則制定的賽事中重現？

## 正式 v4 盲測納入條件

- 預測在開賽前建立。
- `系統版本` 以 `4` 開頭。
- `鎖定狀態 = 已鎖定`。
- 有完整 `預測內容雜湊`、版本與鎖定時間。
- 實際比分在之後回填，且只含 90 分鐘。

舊案例、缺少鎖定時間的資料與規則制定樣本可以展示，但不計入 v4 前瞻績效。

## 分層比較

| 層 | 輸出 | 對照 |
|---|---|---|
| 足球基線 | λ、勝平負機率、三比分 | 無卦象 |
| 有界卦象 λ | 修正後 λ 與 Poisson 網格 | 足球基線 |
| 連續卦象劇本 | 閘門、情境混合後機率與三比分 | 有界卦象 λ |
| AI | 候選池內三比分與連續敘事 | 固定劇本 |
| 最終 | 受控融合三比分 | 所有前層 |

## 指標

- 首選精確比分命中率。
- 前三精確比分命中率。
- 首選勝平負命中率。
- Manhattan 比分距離：`|預測體球-實際體球| + |預測用球-實際用球|`。
- 總進球絕對誤差。
- BTTS 與大小 2.5 命中。
- 三分類 Brier score：越低越好。
- LogLoss：越低越好，會懲罰對錯誤方向的過度自信。

精確比分命中適合展示，但不應作為唯一模型選擇指標；小樣本時優先看 Brier、LogLoss、方向與比分距離。

## 規則開發與留出

1. 先凍結規則版本與評估指標。
2. 開發樣本只用來提出假說。
3. 留出樣本在結果揭曉前不得參與規則或參數選擇。
4. 預先指定最少樣本與升級條件。
5. 報告所有結果，不只報告命中案例。
6. 新規則若只改善單一精確比分、卻惡化 Brier 或其他方向，不能升級。
7. 4.0 與 4.1 必須分版本報告；384 組結構測試只證明輸出契約與多樣性，不代表預測準確。

## 建議報告區間

- 少於 20 場：只報原始數字與個別案例，不下穩定結論。
- 20–49 場：報命中率與平均誤差，明確標註探索性。
- 50 場以上：增加信賴區間、版本分層與對手強度分層。
