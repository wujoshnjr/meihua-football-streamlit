# Changelog

## v5.7.1 — 三格一鍵清除與送出前檢查

- 體方自述、用方自述及賽前中性介紹各新增獨立一鍵清除按鈕，只清除指定欄位。
- 清除輸入時同步移除上一筆畫面排卦結果，避免新內容旁仍顯示舊卦。
- 新增「只檢查格式與計數」，可先查看三段實際計數及完整錯誤，不會起卦或儲存紀錄。
- 表單停用 Enter 自動提交，避免游標換行時誤觸第一個清除按鈕。

## v5.7.0 — 球隊第一人稱起象輸入 v2

- 三個輸入區正式改名為「體方自述（起象）」、「用方自述（起象）」與「賽前中性介紹（動爻）」。
- 體、用自述固定使用第一人稱九項結構與「我是隊名」開頭，依系統計數須為 180～220 數。
- 中性介紹固定使用第三人稱、同時提及雙方，依系統計數須為 300～450 數。
- 表單在排卦前檢查人稱、固定結構、雙方名稱及字數範圍，避免大量球員細節沖淡同一股起象之氣。
- HTML／Markdown 報表、後台 JSON 與 CSV 同步標記 v2 輸入用途及三段新名稱；舊 CSV 欄名讀取時自動遷移。
- 保持賽前資訊、九十分鐘範圍及完整排卦用途，不恢復 AI 或比分預測。
- 資料結構版本升至 5.6。

## v5.6.0 — 條件式卦義庫

- 八個經卦各建立八個可選義項與六條條件規則，共 64 個義項、48 條規則。
- 條件訊號固定由體用生克、旺衰、動爻早晚、旬空、月破、六沖六合與整條卦線的破門通道產生。
- 同一卦可以同時保留正反多義；系統只排序前三個優先義項，規則分不是機率、比分或勝負信心。
- 完整落實艮的封鎖、停滯、守成、防守穩定、攻勢中斷、控制節奏、單點屏障與被困等八義，以及六組指定判斷條件。
- 排卦畫面、HTML 完整表、後台 JSON 與可讀 CSV 同步加入體用條件式卦義與命中條件。
- 卦象資料庫改用可讀表格呈現全部義項與條件，不再以 JSON 方塊顯示經卦資料。
- 資料結構版本升至 5.5。

## v5.5.0 — 完整卦義、納甲、世應、六親與可讀下載表

- 六十四卦各新增六項完整卦義及九項足球比賽應用參考，並明確與經典原文分層。
- 起卦時間新增日辰、日干、日支；依六十甲子自動計算旬名及兩個旬空地支。
- 本卦、互卦、變卦逐爻套用固定納甲表，列出天干地支、五行、旬空與動爻標記。
- 依八宮卦序自動安世應；六親同時輸出使用者指定的日干法與常見卦宮法。
- 檢出爻間、動爻及世應的地支六沖六合，足球說明只作非決定性的應用參考。
- 排卦下載改為完整 HTML 表，紀錄 CSV 隱藏原始 JSON 與排卦指紋；後台仍保留完整 JSON 稽核資料。
- 資料結構版本升至 5.4。

## v5.4.1 — 經傳、動爻結構與月令旺衰 JSON

## v5.1.1 — Community Cloud 原生崩潰第二層修復
