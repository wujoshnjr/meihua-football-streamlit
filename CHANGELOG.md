# Changelog

## v5.4.0 — 焦氏易林完整標點與 JSON 輸出

- JSON 新增主卦與變卦的卦辭、彖傳、大象，以及動爻爻辭與小象。
- JSON 新增動爻得位、得中、相應、乘、承、比等可稽核結構欄位。
- 依農曆月建固定計算體用動爻前後的旺相休囚死與強弱移動；時支五行另列且不覆寫月令。
- 將 4,096 條《焦氏易林》林辭升級為繁體標點版，並新增逐條標點完整性驗證。
- 標點顯示本採維基文庫校對文本；其三條原缺項目由 Kanripo 四庫本補足並留下可稽核記錄。
- 排卦下載 JSON 新增 `jiaoshi_yilin`，包含本卦之卦索引、完整林辭、卦序結構、版本與授權來源。
- 後台保存的「完整排盤JSON」同步加入同一筆焦氏易林內容，並保留既有平面排盤欄位相容性。
- 資料結構版本升至 5.3，知識庫版本升至 `zhouyi-64x384-yilin4096-punctuated-v1.2.0`。

## v5.3.0 — 完整焦氏易林資料庫

- 新增《焦氏易林》64 本卦 × 64 之卦，共 4,096 條非空林辭。
- 排卦結果與 Markdown 報告自動附上「本卦之變卦」的原典林辭，但不解釋、不參與取數。
- 卦象資料庫新增本卦／之卦雙選擇器，可獨立查閱全部林辭。
- 啟動完整性檢查加入 64×64 覆蓋、空白林辭與卦序一致性驗證。
- 收錄 Kanripo《欽定四庫全書》底本快照、CC0 來源資訊及一筆來源標題校正紀錄。

## v5.2.1 — 移除補充資料輸入框

- 表單只保留體方段落、用方段落與完整中性段落三個文字框。
- 新排盤報告不再輸出空白的補充資料段落。
- CSV 的舊補充資料欄位僅為歷史紀錄相容而保留，新紀錄固定留空。

## v5.2.0 — 起卦農曆時間

- 起卦時固定記錄台北國曆時間、完整農曆年月日、干支年與干支時辰。
- 畫面、JSON、Markdown 與 CSV 共用同一份時間快照，且時間不參與文字取數。

## v5.1.1 — Community Cloud 原生崩潰第二層修復

- 應用啟動路徑移除 pandas、NumPy、PyArrow 與 openpyxl 直接載入。
- 排卦紀錄改用 Python 標準庫 CSV，紀錄下載由 Excel 改為 UTF-8 CSV。
- Streamlit 表格改為安全 HTML，避免健康檢查時啟動 Arrow 資料轉換。
- 固定 pandas 2.2.3、NumPy 2.2.6 與 PyArrow 19.0.1，避免 `uv` 重建時原生套件漂移。
- Community Cloud 停用 Streamlit 內建檔案監看，由平台自身負責 GitHub 更新。

## v5.1.0 — 隊伍名稱自動組合

- 事件／比賽名稱改為左右兩個隊伍輸入框，中央 `vs` 固定顯示。
- `vs` 前方名稱自動作為體方，後方名稱自動作為用方，事件名稱同步組合，不再重複輸入。
- 新增體用名稱必填與自動組合測試。

## v5.0.1 — Streamlit Community Cloud 啟動修正

- 固定 Streamlit 1.56.0，避開新版 Starlette／Uvicorn 啟動後的原生程序崩潰。
- 新增真實 Streamlit CLI 與 `/_stcore/health` 啟動測試，不再只依賴 AppTest。

## v5.0.0

- 產品由足球比分預測系統改為「完整排卦系統」。
- 移除 AI、足球先驗、比分網格、規則校準、命中率、相似案例與賽後流程。
- 輸入保留體方段落、用方段落與完整中性段落；補充資料只保存、不取數。
- 輸出新增可稽核計數、餘數、本卦六爻表、互卦取爻、動爻陰陽反轉與變卦完整結構。
- 新增 JSON／Markdown 排盤下載與獨立 `meihua_castings.csv` 儲存。
- 八卦資料庫改為傳統結構欄位，移除足球攻守與進球欄位。
- 六十四卦資料庫擴充為 64 卦、384 爻全覆蓋，包含卦辭、彖傳、大象、爻辭與小象。
- 乾坤加入用九／用六與《文言》，另加入《說卦》《繫辭》《序卦》《雜卦》。
- 每卦新增互卦、錯卦與綜卦索引。
